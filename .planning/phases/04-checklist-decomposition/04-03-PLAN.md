Rewrite `lib/features/inspection/presentation/form_checklist_page.dart` to be a thin orchestrator.

**Keep the same public API** — the FormChecklistPage constructor signature stays identical (InspectionDraft + optional service params) so that the GoRouter route and existing tests don't break at the constructor level.

**State changes:**
- Replace all 19 mutable state fields with a single `late InspectionSessionController _controller`
- Add `int _activeTabIndex = 0` for tab selection
- Remove `_mediaCapture` field (moved to controller)
- Remove all business logic methods (moved to controller)
- Remove all private build helpers (moved to sub-views)

**initState:**
```dart
@override
void initState() {
  super.initState();
  _controller = InspectionSessionController(
    draft: widget.draft,
    repository: widget.repository,
    signatureRepository: widget.signatureRepository,
    // ... pass through all service dependencies
  );
  _controller.onStateChanged = () {
    if (mounted) setState(() {});
  };
  _controller.initialize();
}
```

**build method:**
```dart
@override
Widget build(BuildContext context) {
  return Scaffold(
    appBar: AppBar(title: const Text('Guided Inspection Wizard')),
    body: ListView(
      padding: const EdgeInsets.all(16),
      children: [
        // Header: client name + property address
        Text('Inspection for ${widget.draft.clientName}', ...),
        Text(widget.draft.propertyAddress),
        const SizedBox(height: 16),

        // Tab selector (SegmentedButton with 4 segments)
        _buildTabSelector(),
        const SizedBox(height: 16),

        // Active sub-view
        _buildActiveView(),
      ],
    ),
  );
}
```

**Tab selector** — use Material 3 `SegmentedButton<int>` with 4 segments:
- 0: "Steps" (Icons.list_alt)
- 1: "Summary" (Icons.assignment)
- 2: "Report" (Icons.picture_as_pdf)
- 3: "Timeline" (Icons.history)

**Active view rendering** — switch on `_activeTabIndex`:
- 0: `WizardNavigationView(...)` with callbacks wired to controller methods + SnackBar feedback
- 1: `EvidenceCaptureView(formSummaries: _controller.wizardState.buildFormSummaries())`
- 2: `PdfDeliveryView(...)` with callbacks wired to controller methods + SnackBar feedback
- 3: `AuditTimelineView(auditEvents: _controller.auditEvents, ...)`

**Callback wiring with SnackBar feedback:**
For methods that previously showed SnackBars directly, the parent now handles feedback based on return values:

```dart
void _handleContinue() async {
  final result = await _controller.continueStep();
  if (!mounted) return;
  switch (result) {
    case ContinueStepResult.blocked:
      // Show snackbar with missing items
      break;
    case ContinueStepResult.finished:
      ScaffoldMessenger.of(context).showSnackBar(
        const SnackBar(content: Text('Inspection wizard complete.')),
      );
      break;
    // etc.
  }
}
```

Similarly for `_handleGeneratePdf()`, `_handleDownload()`, `_handleShare()`.

**Critical**: Preserve ALL `ValueKey` strings from the original monolith in the sub-views (already done in Plan 04-02). The parent doesn't need ValueKeys itself — they're in sub-views.

**Critical**: The `FormChecklistPage` widget class must keep the same constructor interface so `router_config.dart` doesn't need changes beyond cosmetic. Verify the constructor still accepts `InspectionDraft draft` required + all optional service params.
