Create `lib/features/inspection/presentation/controllers/inspection_session_controller.dart`.

This is a **plain Dart class** (not a Widget, not a ChangeNotifier). It extracts all business logic and mutable state from `_FormChecklistPageState`.

**Constructor** accepts:
- `InspectionDraft draft` (required)
- All 10 service dependencies as named parameters (same pattern as current FormChecklistPage constructor — optional with `.live()` defaults):
  - `InspectionRepository? repository`
  - `SignatureRepository? signatureRepository`
  - `ReportSignatureEvidenceRepository? signatureEvidenceRepository`
  - `DeliveryService? deliveryService`
  - `MediaSyncRemoteStore? mediaSyncRemoteStore`
  - `PendingMediaSyncStore? pendingMediaSyncStore`
  - `PdfOrchestrator? pdfOrchestrator`
  - `CloudPdfService? cloudPdfService`
  - `AuditEventRepository? auditRepository`
  - `MediaCaptureService? mediaCapture` (NOTE: currently created inline as `_mediaCapture = MediaCaptureService()` in the monolith — must be promoted to injectable dependency for testability)

**State fields** (public getters, private setters):
- `WizardProgressSnapshot snapshot`
- `int currentStepIndex`
- `bool isGenerating`
- `bool isSavingProgress`
- `String? lastPdfPath`
- `ReportReadiness? persistedReadiness`
- `ReportArtifact? lastArtifact`
- `List<AuditEvent> auditEvents`
- `bool isLoadingAuditEvents`
- `String? auditTimelineError`

**Computed getters:**
- `InspectionWizardState get wizardState` — same as current `_wizardState` getter
- `ReportReadiness get effectiveReadiness` — returns `persistedReadiness ?? _evaluateReadiness()`

**Callback hook:**
- `VoidCallback? onStateChanged` — parent sets this to `() => setState(() {})`. Controller calls it after every state mutation.

**Public async methods** — extract directly from `_FormChecklistPageState`:
- `Future<void> initialize()` — calls `_hydrateCapturedFromSnapshot()`, sets `currentStepIndex`, calls `_loadReadiness()` and `_loadAuditEvents()`. Called once by parent in initState.
- `Future<void> continueStep()` — from `_continueStep()`. Returns a `ContinueStepResult` enum (advanced, finished, blocked, error) instead of showing SnackBars directly.
- `Future<CaptureResult> capture(EvidenceRequirement requirement)` — from `_capture()`. Returns result instead of showing SnackBars.
- `Future<PdfGenerationResult> generatePdf()` — from `_generatePdf()`. Returns result enum (success, cloudTerminalFailure, error) with optional error message.
- `Future<DeliveryResult> downloadArtifact()` — from `_downloadLastArtifact()`. Returns result with URL or error.
- `Future<DeliveryResult> shareArtifact()` — from `_shareLastArtifact()`. Returns result with URL or error.
- `Future<void> loadAuditEvents()` — from `_loadAuditEvents()`.
- `void setBranchFlag(String key, bool value)` — from `_setBranchFlag()`.

**Result types** — define simple enums/sealed classes at the top of the file or in a separate small file:
- `enum ContinueStepResult { advanced, finished, blocked, error }`
- `enum CaptureResult { captured, cancelled }`
- `class PdfGenerationResult { final bool success; final String? errorMessage; final bool isCloudTerminalFailure; }`
- `class DeliveryResult { final bool success; final String? url; final String? errorMessage; }`

**Private helper methods** — move from `_FormChecklistPageState`:
- `_loadReadiness()`, `_syncReadinessFromSnapshot()`, `_evaluateReadiness()`
- `_hydrateCapturedFromSnapshot()`, `_categoryForRequirementKey()`
- `_saveProgress()`, `_loadEvidenceMediaPaths()`, `_mergeEvidencePaths()`, `_missingCompletedEvidenceKeys()`

**Critical**: The controller must NOT import Flutter (`package:flutter/material.dart`). It is pure Dart. No BuildContext, no ScaffoldMessenger, no mounted checks. The parent page handles all UI feedback (SnackBars) based on return values from controller methods.

**Critical**: The PdfOrchestrator initialization logic from `initState()` must be handled in the controller's `initialize()` method, since it has conditional setup logic with `remoteStore`. The state machine for PDF generation follows this flow:
```
isGenerating=true → cloud attempt → success → set lastArtifact, isGenerating=false
                                  → PdfCloudGenerationTerminalFailure → return cloudTerminalFailure result
                   → other error → return error result with message
```
Document this state machine as a comment block above `generatePdf()` in the controller.

**Static maps** `_branchFlagsByForm` and `_branchFlagLabels` move from `_FormChecklistPageState` to the controller (they are data, not UI).
