## Task 1: Create WizardProgressIndicator Widget

**Action**: Create `lib/common/widgets/wizard_progress_indicator.dart` — a composite widget showing step progress with a linear bar and percentage.

```dart
class WizardProgressIndicator extends StatelessWidget {
  const WizardProgressIndicator({
    super.key,
    required this.currentStep,
    required this.totalSteps,
    required this.completionPercent,
  });

  final int currentStep;
  final int totalSteps;
  final int completionPercent;

  @override
  Widget build(BuildContext context) {
    final theme = Theme.of(context);
    return Column(
      crossAxisAlignment: CrossAxisAlignment.stretch,
      children: [
        Row(
          mainAxisAlignment: MainAxisAlignment.spaceBetween,
          children: [
            Text(
              'Step $currentStep of $totalSteps',
              style: theme.textTheme.titleSmall,
            ),
            Text(
              '$completionPercent%',
              style: theme.textTheme.titleSmall?.copyWith(
                color: theme.colorScheme.primary,
                fontWeight: FontWeight.w700,
              ),
            ),
          ],
        ),
        const SizedBox(height: AppSpacing.spacingXs),
        ClipRRect(
          borderRadius: AppRadii.sm,
          child: LinearProgressIndicator(
            value: completionPercent / 100,
            minHeight: AppSpacing.spacingSm,
          ),
        ),
      ],
    );
  }
}
```

## Task 5: Add completionPercent Getter to Controller

**Action**: Add a `completionPercent` getter to `InspectionSessionController` in `lib/features/inspection/presentation/controllers/inspection_session_controller.dart`.

CRITICAL: The `snapshot.completion` map ONLY contains entries for requirements that have been captured (set to `true`). Requirements that haven't been interacted with are NOT in the map. Therefore, using `completionMap.length` as the denominator gives WRONG results (e.g., 3 captured out of 10 total → map has 3 entries → 3/3 = 100% WRONG).

The correct denominator is the TOTAL visible requirement count from all wizard steps:

```dart
/// Percentage of visible requirements that have been captured (0-100).
int get completionPercent {
  // Total visible requirements across all wizard steps
  final totalRequirements = wizardState.steps
      .expand((step) => step.requirements)
      .length;
  if (totalRequirements == 0) return 0;

  // Count captured requirements from completion map
  final captured = snapshot.completion.values.where((v) => v == true).length;
  return ((captured / totalRequirements) * 100).round();
}
```

This uses `wizardState` which is a computed getter already on the controller. The `wizardState.steps` list contains all visible steps, and each step has a `requirements` list. This gives the correct total count.

**Verify**: `dart analyze` — 0 errors. Test with 0 requirements → 0%. Test with 3/10 captured → 30%. Test with 10/10 → 100%.
**Done**: Controller has `completionPercent` getter with correct denominator from wizardState.steps.

Update `test/features/inspection/controllers/inspection_session_controller_test.dart`:

- Test: completionPercent returns 100 when all requirements captured
