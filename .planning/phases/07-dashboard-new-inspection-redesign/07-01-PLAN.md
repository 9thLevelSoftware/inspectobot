<task type="auto">
  <name>Task 2: Implement inspection cards with visual status badges</name>
  <files>lib/features/inspection/presentation/dashboard_page.dart</files>
  <action>
Design the inspection list items as styled cards with visual status indicators.

**IMPORTANT — Use SectionCard for custom card layout** (not InspectionCard, which may not support StatusBadge rendering). Build the card content manually inside SectionCard.

**Inspection card design:**
Each inspection in the list should be rendered as a `SectionCard` containing:

1. **Header row**: Client name (titleMedium, bold) + StatusBadge on the right
   - StatusBadge variants based on WizardProgressStatus:
     - `inProgress` → StatusBadge(label: "In Progress", type: StatusBadgeType.warning, highContrast: true)
     - `complete` → StatusBadge(label: "Complete", type: StatusBadgeType.success, highContrast: true)
   - **Draft status derivation**: If the inspection has NO WizardProgressSnapshot (snapshot is null), OR if snapshot.lastStepIndex == 0 with zero completions, treat it as "Draft" → StatusBadge(label: "Draft", type: StatusBadgeType.neutral, highContrast: true)
   - Do NOT add a new enum value to WizardProgressStatus — derive draft from absence of meaningful progress

2. **Body**: Property address (bodyMedium, onSurfaceVariant color)

3. **Footer row**: Inspection date (timestamp style) + Resume/View action button
   - In-progress: `AppButton` variant="outlined" label="Resume" (navigates to checklist at resume step)
   - Complete: `AppButton` variant="text" label="View"

4. **Tap target**: Entire card should be tappable (InkWell/GestureDetector) to navigate to the checklist — min 48dp height per AppFieldUsability.minTapTarget

5. **Card spacing**: `AppSpacing.spacingMd` gap between cards in the list

**Status derivation logic:**
Create a helper method `_inspectionStatus(InspectionWizardProgress progress)` that returns the appropriate status based on the wizard snapshot's status field. Keep this as a simple presentation-layer mapping — no domain model changes.
