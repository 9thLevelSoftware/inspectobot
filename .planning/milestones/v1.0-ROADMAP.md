## Overview

InspectoBot v1.0 MVP delivers a compliance-first Florida insurance inspection app for independent inspectors. End-to-end workflow from secure onboarding, through field inspection wizard execution with offline tolerance, evidence capture enforcement, official-form-aligned PDF generation, to legally defensible delivery with immutable audit trails and retention controls.

## Milestone Summary

**Key Decisions:**
- `docs/blueprint.md` is canonical product intent
- On-device PDF generation is default architecture direction
- Required-photo gating is a hard blocker for PDF generation
- Auth repository + AuthGate split keeps Supabase session logic out of page widgets
- Offline capture and idempotent sync are treated as a distinct delivery boundary
- Form rendering is manifest-driven with pinned revision+map versions
- PDF size guardrails use bounded retries and explicit over-budget failure metadata
- Audit timeline persistence is append-only and fail-closed
- Delivery links are short-lived signed URLs with persisted delivery action rows
- Retention policies enforce a minimum five-year floor
- Cloud PDF fallback uses explicit generated/unavailable/terminal outcomes
- License source keys are non-required unless policy is intentionally changed
- Canonical branch keys are defined in FormRequirements and reused by predicates/tests
- Coverage enforcement tests compare canonicalSourceKeysForForm against map source_key sets

**Issues Resolved:**
- Password recovery deep-link not wired from email callback to in-app reset (Phase 13)
- Resume flow dropping evidence media references before PDF mapping (Phase 14)
- Media sync metadata and PDF resolver remote-key mismatch (Phase 14)
- Audit timeline writes with no UI consumer (Phase 15)
- Cloud PDF fallback returning null placeholder (Phase 15 + 18)
- 11 orphaned requirement IDs not backed by verification evidence (Phase 16)
- Tenant auth/isolation flows partially verified with human_needed status (Phase 17)
- Conditional branching not wired to live user inputs (Phase 19)
- ~75% evidence keys missing from PDF field maps (Phase 20)
- Inspector license/profile contract ambiguous (Phase 20)

**Issues Deferred:**
- None. All identified gaps closed.

**Technical Debt Incurred:**
- None remaining. All v1.0 tech debt items resolved by phases 15, 18, and 20.
