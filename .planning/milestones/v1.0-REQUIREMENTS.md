## v1.0 Original Requirements (26/26)

### Authentication & Security

- [x] **AUTH-01**: Inspector can create an account with email/password and sign in successfully. — *Phase 1, verified Phase 17*
- [x] **AUTH-02**: Inspector remains signed in across app restarts and can recover access. — *Phase 1, verified Phase 17*
- [x] **AUTH-03**: Inspector can recover access via password reset email flow with in-app completion. — *Phase 1, hardened Phase 13*
- [x] **AUTH-04**: Inspector can save inspector profile details including license type and number. — *Phase 1, reconciled Phase 20 (AUTH-04 POLICY: non-consumer)*
- [x] **AUTH-05**: Inspector can save and update stored signature for later report signing. — *Phase 1, verified Phase 16*
- [x] **SEC-01**: Inspector cannot access inspection data belonging to another organization. — *Phase 1, live-verified Phase 17*
- [x] **SEC-02**: Signed reports store signature evidence metadata (timestamp, hash linkage, attribution). — *Phase 6, verified Phase 16*

### Workflow

- [x] **FLOW-01**: Inspector can create a new inspection with client, property, date, and year-built context. — *Phase 2, verified Phase 17*
- [x] **FLOW-02**: Inspector can select one or more supported Florida forms for the inspection. — *Phase 2, verified Phase 16*
- [x] **FLOW-03**: Inspector can complete a linear wizard with required-step progression and conditional branching. — *Phase 3, wired Phase 19*
- [x] **FLOW-04**: Inspector can leave and later resume an in-progress inspection at the last incomplete step. — *Phase 3, hardened Phase 14*
- [x] **FLOW-05**: Inspector can view per-form completion summaries identifying missing required items. — *Phase 3, verified Phase 16*

### Evidence Capture

- [x] **EVID-01**: Inspector can capture required exterior elevation photos. — *Phase 5, verified Phase 16*
- [x] **EVID-02**: Inspector can capture required roof evidence including slope-specific and defect photos. — *Phase 5, activated Phase 19*
- [x] **EVID-03**: Inspector can capture required plumbing, HVAC, electrical, and hazard photos. — *Phase 5, activated Phase 19*
- [x] **EVID-04**: Inspector can capture evidence for all seven wind mitigation categories. — *Phase 5, activated Phase 19*
- [x] **EVID-05**: System blocks PDF generation when required fields or evidence are incomplete. — *Phase 6, verified Phase 16*
- [x] **EVID-06**: Captured images are compressed to mobile-friendly sizes before upload and PDF embedding. — *Phase 5, verified Phase 16*

### PDF Generation

- [x] **PDF-01**: System generates official-form-aligned PDFs using version-pinned templates. — *Phase 7, integrity-hardened Phase 11*
- [x] **PDF-02**: Generated PDFs include mapped data, mapped images, and inspector signature in required locations. — *Phase 7, coverage-completed Phase 20*
- [x] **PDF-03**: Generated PDF artifacts stay within configurable size guardrails. — *Phase 7*

### Delivery & Data

- [x] **DLV-01**: Inspector can deliver generated reports via download and secure sharing. — *Phase 8, durability-hardened Phase 12*
- [x] **DLV-02**: System records an immutable audit timeline for inspection actions. — *Phase 8, UI-exposed Phase 15, durability Phase 12*
- [x] **DATA-01**: Report records remain accessible per configured retention controls (5-year baseline). — *Phase 8, verified Phase 16*

### Offline

- [x] **OFF-01**: Inspector can complete core capture workflows while offline. — *Phase 4, media-rehydrated Phase 14*
- [x] **OFF-02**: Sync retries do not create duplicate or orphan records. — *Phase 4, dependency-enforced Phase 12*
